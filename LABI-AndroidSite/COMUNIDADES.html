XDA Developers


Criada em janeiro de 2013 pela empresa NAH6 Crypto Products BV, a XDA Developers é a maior comunidade de desenvolvimento de software móvel do mundo com cerca de 6,9 milhões de membros, sendo também das mais ativas devido aos seus 2,8 milhões de tópicos e 60,4 milhões de mensagens. Embora a XDA Developers esteja aberta a falar sobre determinados sistemas operativos móveis, é maioritariamente virada para o sistema operativo móvel da Google e é lá onde se encontram a esmagadora maior parte os desenvolvedores e programadores para Android. É nesta comunidade onde são discutidos e desenvolvidos métodos para a resolução de problemas sobre determinados sistemas móveis, oferecendo, também, ferramentas para quem quer costumizar o seu smartphone/tablet e outros equipamentos móveis, como por exemplo, Custom ROMs, atualizações de ROMs, Root ou Super User, Custom Recovery’s, Custom Kernels e por aí em diante. A comunidade oferece informações gerais sobre dispositivos, atualizações de ROMs, suporte técnico e revisões de aplicações e acessórios.


Google Plus


Google Plus é uma rede social da Google que pretende facilitar a fácil partilha de conhecimento através da web. Cada utilizador possui um perfil próprio, onde pode seguir outros utilizadores, páginas ou ingressar em várias comunidades sobre qualquer assunto. No entanto esta rede social é maioritariamente usada por entusiastas de Android. Existem diversas comunidades em volta do Android que são usadas para a troca de conhecimento em programação Android ou para servir de ponte para projetos de outras comunidades, encontrando-se diversos testers nesta rede.


Git e Github


Git é um sistema Open Source de controlo de versões, conhecido pela sua velocidade e estabilidade. Foi criado em 2006 para gerir todo o Linux Kernel e tem agora um conjunto abrangente de recursos, como uma equipa de desenvolvimento ativa e muitas comunidades de hospedagem gratuitas. O Git tem uma extrema importância no ecossistema Android, pois é através dele que todo este projeto é desenvolvido, atualizado e modificado, distribuindo-se através dos seus múltiplos repositórios. O Android Open Source Project (AOSP) utiliza “code lines” em vez de “branches”, porque a qualquer momento pode haver mais que um ramo numa determinada “linha de código”, podendo tambem “tornar-se num novo ramo com base nas necessidades num dado momento”, de acordo com a Google. São os Engenheiros da AOSP quem gerem e que fazem a manutenção de toda esta série de “code lines” de forma a que o projecto siga uma linha de versões estáveis e atualizadas, formando uma rede de branches em forma de árvore (tree). Esta árvore é atualizada por novas versões Android. A necessidade do Git é explicada pela Google: “Queríamos que os componentes fossem substituíveis, e queríamos componentes interessantes para serem capazes de crescer numa vida própria, fora do Android. Nós escolhemos primeiro um sistema de controlo de revisão distribuído, para, em seguida, reduzi-lo ainda mais ao Git.” Como se não bastasse, a Google ainda criou outra ferramenta de gerenciamento de repositório construída sobre o Git, chamado de Repo. O Repo junta diversos repositórios Git em momentos necessários, é usado como um meio de transmissão para o sistema de controlo de revisão da Google e analiza o fluxo de desenvolvimento Android. O Repo é usado para simplificar o uso do Git no contexto Android. Com origem num script Python executável, os comandos Repo são extremamente úteis para quem quer transferir múltiplos arquivos ao mesmo tempo de vários repositórios. A relação Git e Repo, de acordo com a Google, é exemplificada: “Na maioria das situações, o utilizador pode usar o Git em vez de Repo, ou misturar Repo e comandos Git para formar comandos complexos. Usando Repo para operações básicas em toda a rede, o utilizador fará um trabalho muito mais simples”. Gerrit é um sistema de revisão de código baseado na Web para projetos que usam git, o que torna mais fácil rever, exibindo mudanças lado a lado no navegador e permitindo comentários in-line. Android Studio é o “ambiente de desenvolvimento integrado oficial” (IDE) para desenvolvimento 5de aplicações Android. É no Android Studio onde são editados os futuros commits. O Github é um serviço web que oferece diversas funcionalidades extras aplicadas ao Git, como feeds, followers, wiki e um gráfico que mostra como os desenvolvedores trabalham as versões dos seus repositórios. Pode se fazer uso deste serviço de forma gratuita para hospedar projetos pessoais. Além disso, quase todos os projetos sobre desenvolvimento Open Source estão no Github, e é possível acompanhá-los através de novas versões, contribuir informando bugs ou até mesmo enviando código e correções. É dos repositórios mais usados pela comunidade Android.

Outras Comunidades


Existem outras comunidades importantes como o BasketBuild, Dev-Host, Android File Host e o Google Code (ou Google Developers), que são importantes para a hospedagem de diversos ficheiros como GApps (Aplicações da Google), ROMs, Kernels, Aplicações e mods. Estas comunidades são importantes para a troca de ficheiros que constituem o Android Open Source Project.
